# Framelink MCP 工具参考

Framelink MCP for Figma 的完整工具参考文档。

## 概述

Framelink MCP 是一个专为 AI 编码代理（如 Cursor）设计的 Model Context Protocol 服务器。它从 Figma API 获取设计数据，并简化响应以提供最相关的布局和样式信息。

**核心优势**：

- 压缩 Figma API 响应约 90%
- 只返回 AI 需要的关键设计信息
- 专为一次性准确实现设计优化

## 可用工具

### 1. get_figma_data

**Figma 设计转代码的主要工具。**

#### 用途

从 Figma 文件、帧或组获取简化后的设计上下文，为代码生成提供必要的布局和内容信息。

#### 使用方式

1. 在 Figma 中右键点击帧或组
2. 选择「复制/粘贴为」>「复制选择项的链接」
3. 在 Cursor 中粘贴链接
4. 请求 AI 实现设计

#### 返回内容

- 布局结构（flex、grid、定位）
- 尺寸（宽度、高度）
- 间距（margin、padding、gap）
- 颜色（背景、文字、边框）
- 排版（字号、字重、行高）
- 圆角和阴影
- 图片和图标的资源信息

---

### 2. download_figma_images

**下载 Figma 文件中的图片资源。**

#### 用途

根据图片或图标节点的 ID，下载 Figma 文件中使用的 SVG 和 PNG 图片。

#### 参数

| 参数        | 类型   | 必填 | 说明                   |
| ----------- | ------ | ---- | ---------------------- |
| `fileKey`   | string | 是   | Figma 文件的 key       |
| `nodes`     | array  | 是   | 要下载的节点数组       |
| `localPath` | string | 是   | 保存图片的本地绝对路径 |
| `pngScale`  | number | 否   | PNG 导出比例，默认 2   |

#### nodes 数组项

| 参数       | 类型   | 必填 | 说明                                     |
| ---------- | ------ | ---- | ---------------------------------------- |
| `nodeId`   | string | 是   | 节点 ID，格式如 `1234:5678`              |
| `fileName` | string | 是   | 保存的文件名，需以 `.png` 或 `.svg` 结尾 |
| `imageRef` | string | 否   | 如果节点有 imageRef 填充，需包含此参数   |

#### 使用场景

- 下载设计稿中的图标（SVG）
- 下载设计稿中的图片（PNG）
- 批量下载多个资源

## 工作流程模式

### 模式 1：简单组件

```
1. 粘贴 Figma 帧链接
2. 请求实现设计
3. AI 获取设计数据并生成代码
4. 验证结果
```

### 模式 2：包含图片的设计

```
1. 粘贴 Figma 帧链接
2. AI 获取设计数据
3. AI 调用 download_figma_images 下载图片资源
4. 生成引用这些图片的代码
```

### 模式 3：复杂页面

```
1. 将页面分解为逻辑区块
2. 对每个区块分别获取设计数据
3. 下载所需的图片资源
4. 逐区块实现并验证
```

## 常见问题

### 「权限被拒绝」

- **原因**：未认证或无文件访问权限
- **解决**：检查 Figma 访问令牌是否有效且具有正确权限

### 「节点未找到」

- **原因**：无效的 nodeId 或 fileKey
- **解决**：检查 URL 解析，确保节点在 Figma 中存在

### 「输出过大」

- **原因**：节点包含太多内容
- **解决**：分解为更小的区块，一次处理一个
