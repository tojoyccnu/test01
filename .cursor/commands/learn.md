# /learn - 经验沉淀

从当前会话中提炼可复用的经验，保存到 learned skill 供后续使用。

## 何时使用

解决了有一定复杂度的问题后，运行 `/learn` 进行经验沉淀。

## 关注哪些内容

1. **问题排查经验** — 错误原因、修复方法、是否可复用
2. **调试技巧** — 不易想到的排查思路、好用的工具组合
3. **踩坑记录** — 第三方库的坑、API 限制、版本特定问题
4. **项目相关经验** — 隐式约定、架构决策、集成模式

## 目录结构

```
.cursor/skills/learned/
├── SKILL.md                    # 主文件（经验索引）
└── references/                 # 所有经验条目
    ├── vue-query-cache.md
    ├── pnpm-peer-deps.md
    └── ...
```

## 首次初始化

如果 `.cursor/skills/learned/` 不存在，先创建目录和 SKILL.md：

```markdown
---
name: learned
description: 开发过程中沉淀的经验和踩坑记录。当遇到类似问题、调试卡住、或需要回顾历史解决方案时自动触发。
---

# 经验库
```

## 新增经验

在 `references/` 目录下创建新文件 `[经验名称].md`：

```markdown
# [简明标题]

**日期：** YYYY-MM-DD
**关键词：** [便于搜索的关键词，逗号分隔]

## 问题

[具体遇到了什么问题]

## 原因

[问题的根本原因]

## 解决方案

[怎么解决的]

## 示例

[代码或操作步骤（如适用）]
```

同时更新 SKILL.md 中的索引表格。

## 执行步骤

1. 回顾本次会话，找出有价值的经验
2. 检查 `.cursor/skills/learned/` 是否存在，不存在则初始化
3. 确定经验名称（简洁、语义化、kebab-case）
4. 在 `references/` 下创建经验文件
5. 更新 SKILL.md 索引表格
6. 让用户确认后保存，并提醒用户保存后可能需要重启 IDE 使经验生效

## 注意

- 小问题不用记（拼写错误、简单语法问题）
- 偶发性问题不用记（临时服务故障等）
- 重点记录能提升效率、避免重复踩坑的经验
- 一个文件只记一件事，保持聚焦
