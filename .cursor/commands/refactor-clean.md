# 重构清理命令

通过测试验证安全地识别和移除死代码：

1. 运行 knip 死代码分析（查找未使用的文件、导出和依赖）

2. 在 `.reports/dead-code-analysis.md` 生成综合报告

3. 按严重程度分类发现的问题：
   - SAFE（安全）：测试文件、未使用的工具函数
   - CAUTION（谨慎）：API 路由、组件
   - DANGER（危险）：配置文件、主入口文件

4. 仅提议安全的删除操作

5. 每次删除前：
   - 运行完整测试套件
   - 验证测试通过
   - 应用更改
   - 重新运行测试
   - 如果测试失败则回滚

6. 显示已清理项目的摘要

**切勿在未先运行测试的情况下删除代码！**
