# 测试覆盖率分析

分析测试覆盖率并生成缺失的测试：

1. 运行带覆盖率的测试：`pnpm test:coverage`

2. 分析覆盖率报告（coverage/coverage-summary.json）

3. 识别低于 80% 覆盖率阈值的文件

4. 针对每个覆盖不足的文件：
   - 分析未测试的代码路径
   - 为工具函数生成单元测试
   - 为 Vue 组件生成组件测试
   - 为关键用户流程生成 E2E 测试

5. 验证新测试通过

6. 展示前后覆盖率对比

7. 确保项目达到 80%+ 整体覆盖率

## 测试重点

- 正常路径场景
- 错误处理
- 边界情况（null、undefined、空值）
- 边界条件
- 用户交互行为
